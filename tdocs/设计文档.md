# 学生个人知识库与日程标注系统 - 设计文档

## 1. 系统架构

```
┌─────────────────┐       HTTP/JSON       ┌─────────────────────┐
│  JavaFX 客户端   │ ◄──────────────────► │  Spring Boot 后端    │
│  (client.jar)   │    REST API + JWT     │  (server.jar:8088)  │
└─────────────────┘                       └──────────┬──────────┘
                                                     │ JPA
                                                     ▼
                                          ┌─────────────────────┐
                                          │   MySQL 8.0         │
                                          │ student_knowledge_db│
                                          └─────────────────────┘
```

**技术栈：**
- 后端：Spring Boot 3.2.5 + Spring Security + Spring Data JPA + JWT
- 前端：JavaFX 21 (纯Java编程式UI，无FXML)
- 数据库：MySQL 8.0
- 构建：Maven 多模块

## 2. 数据库设计 (ER图)

```
t_user (用户表)
├── id (PK, BIGINT, 自增)
├── username (VARCHAR 50, UNIQUE, 学号/用户名)
├── email (VARCHAR 100, UNIQUE)
├── password (VARCHAR 255, BCrypt加密)
├── role (VARCHAR 20, STUDENT/ADMIN)
├── avatar (VARCHAR 255, 可空)
├── created_at (DATETIME)
└── updated_at (DATETIME)

t_category (知识分类表 - 树形结构)
├── id (PK)
├── name (VARCHAR 100)
├── parent_id (FK → t_category.id, 可空=顶级)
├── user_id (FK → t_user.id)
├── sort_order (INT)
├── created_at / updated_at
└── 关系: 自引用(parent_id) → 支持多级分类树

t_tag (标签表)
├── id (PK)
├── name (VARCHAR 50)
├── color (VARCHAR 20, 默认#1890ff)
├── user_id (FK → t_user.id)
├── created_at
└── 约束: UNIQUE(user_id, name)

t_knowledge_entry (知识条目表)
├── id (PK)
├── title (VARCHAR 200, NOT NULL)
├── content (TEXT, Markdown格式)
├── user_id (FK → t_user.id)
├── category_id (FK → t_category.id, 可空)
├── is_pinned (TINYINT, 是否置顶)
├── created_at / updated_at
└── 多对多关联: → t_knowledge_tag → t_tag

t_knowledge_tag (知识-标签关联表, 多对多)
├── entry_id (PK, FK → t_knowledge_entry.id)
└── tag_id (PK, FK → t_tag.id)

t_task (日程任务表)
├── id (PK)
├── title (VARCHAR 200, NOT NULL)
├── description (TEXT)
├── user_id (FK → t_user.id)
├── status (VARCHAR 20: TODO/IN_PROGRESS/DONE/OVERDUE)
├── priority (VARCHAR 20: LOW/MEDIUM/HIGH/URGENT)
├── start_time (DATETIME, 可空)
├── deadline (DATETIME, NOT NULL)
├── completed_at (DATETIME, 可空)
└── created_at / updated_at
```

### 表关系总结

| 关系 | 说明 |
|------|------|
| t_user → t_category | 一对多 (一个用户多个分类) |
| t_user → t_tag | 一对多 (一个用户多个标签) |
| t_user → t_knowledge_entry | 一对多 |
| t_user → t_task | 一对多 |
| t_category → t_category | 自引用 (树形结构) |
| t_category → t_knowledge_entry | 一对多 |
| t_knowledge_entry ↔ t_tag | 多对多 (通过t_knowledge_tag) |

## 3. 后端API接口设计

### 3.1 认证模块 (/api/auth)
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /auth/login | 登录 | 公开 |
| POST | /auth/register | 注册 | 公开 |

### 3.2 知识条目 (/api/knowledge)
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /knowledge | 获取我的笔记列表 | 登录用户 |
| GET | /knowledge/{id} | 笔记详情 | 登录用户 |
| GET | /knowledge/search?keyword=x | 按标题搜索 | 登录用户 |
| GET | /knowledge/category/{catId} | 按分类筛选 | 登录用户 |
| POST | /knowledge | 创建笔记 | 登录用户 |
| PUT | /knowledge/{id} | 修改笔记 | 登录用户 |
| DELETE | /knowledge/{id} | 删除笔记 | 登录用户 |

### 3.3 分类管理 (/api/category)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /category | 平铺列表 |
| GET | /category/tree | 树形结构 |
| POST | /category | 创建分类 |
| PUT | /category/{id} | 修改分类 |
| DELETE | /category/{id} | 删除分类 |

### 3.4 标签管理 (/api/tag)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /tag | 我的标签列表 |
| POST | /tag | 创建标签 |
| PUT | /tag/{id} | 修改标签 |
| DELETE | /tag/{id} | 删除标签 |

### 3.5 日程任务 (/api/task)
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /task | 我的全部任务 |
| GET | /task/status/{status} | 按状态筛选 |
| POST | /task | 创建任务 |
| PUT | /task/{id} | 修改任务 |
| DELETE | /task/{id} | 删除任务 |

### 3.6 统计 (/api/stats)
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /stats/my | 个人统计 | 登录用户 |
| GET | /stats/admin | 系统统计 | ADMIN |

### 3.7 管理员 (/api/admin)
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /admin/users | 用户列表 | ADMIN |
| PUT | /admin/users/{id}/reset-password | 重置密码 | ADMIN |
| DELETE | /admin/users/{id} | 删除用户 | ADMIN |
| DELETE | /admin/knowledge/{id} | 删除违规笔记 | ADMIN |

## 4. 前端界面设计

### 4.1 登录页
- 渐变背景 + 居中白色卡片
- Tab切换: 登录 / 注册
- 输入校验 + 错误提示

### 4.2 主框架布局
- **顶部**: 系统Logo + 当前用户名+角色 + 退出按钮
- **左侧**: 二级菜单导航 (知识库管理/日程管理/数据统计/系统管理)
- **中央**: 内容区(按菜单切换)
- **底部**: 版权信息

### 4.3 功能模块
- 我的笔记: 表格+搜索+增删改查弹窗
- 笔记分类: 表格+CRUD+父子分类选择
- 标签管理: 表格+颜色选择器
- 我的日程: 表格+状态色彩标记+一键完成
- 已完成任务: 筛选DONE状态
- 学习分析: 卡片数据+饼图(分类)+柱状图(任务状态)
- 用户管理(管理员): 用户表格+重置密码+删除
- 系统概况(管理员): 全局统计+图表

## 5. 安全设计

- 密码: BCrypt加密存储
- 认证: JWT Token (24小时有效期)
- 鉴权: Spring Security + RBAC (STUDENT/ADMIN)
- 数据隔离: 每个API校验userId所属权
