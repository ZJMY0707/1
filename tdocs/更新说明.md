# 更新说明 (v1.1)

## Bug修复

### 1. 日程任务无法创建和编辑
- **现象**：新建任务和编辑任务确认后无效果，无错误提示
- **原因**：客户端发送日期格式为 `"yyyy-MM-dd HH:mm:ss"`，服务端 Jackson 默认无法将该格式反序列化为 `LocalDateTime`，请求返回400错误；客户端未检查响应码，静默失败
- **修复**：
  - `server/dto/TaskDTO.java`：deadline 和 startTime 字段添加 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")` 注解
  - `client/view/TaskListView.java`：创建/编辑后检查响应码，非200时弹出错误提示

### 2. 登录失败时客户端崩溃
- **现象**：用户名或密码错误时报 JsonNull 类型转换异常
- **原因**：登录失败时服务端返回 `data: null`，客户端直接调用 `getAsJsonObject("data")` 导致异常
- **修复**：`client/view/LoginView.java`：先检查 code 是否为200，非200时显示 message 错误信息

### 3. 示例用户密码无法登录
- **现象**：data.sql 中的 BCrypt 哈希值不正确，示例账号无法登录
- **修复**：`server/config/DataInitializer.java`：后端启动时自动检测并修复所有用户的密码哈希

### 4. Hibernate 字段类型校验失败
- **现象**：启动时报 `Schema-validation: wrong column type for priority`
- **修复**：`application.yml`：`ddl-auto` 从 `validate` 改为 `update`

## 部署步骤

1. 初始化数据库：
   ```
   mysql -u root -p123456w --default-character-set=utf8mb4 < sql\schema.sql
   mysql -u root -p123456w --default-character-set=utf8mb4 < sql\data.sql
   ```
2. 构建：`cd server && mvn clean package -DskipTests && cd .. && cd client && mvn clean package -DskipTests && cd ..`
3. 启动后端：`java -jar server\target\server-1.0.0.jar`
4. 启动客户端：`java -jar client\target\client-1.0.0.jar`
5. 测试账号：admin/123456（管理员）、20210001/123456（学生）
